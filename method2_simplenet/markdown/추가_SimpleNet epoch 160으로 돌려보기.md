**느낀점 및 인사이트**

**1. 논문 수치 재현의 어려움**

두 모델 모두 공식 GitHub 코드를 그대로 사용했음에도 불구하고, deprecated 코드 패치, transform 오류 수정, GPU 환경 설정 등 실행 환경을 맞추는 과정 자체가 쉽지 않았다. 논문에서 제시한 수치를 재현하는 것이 단순히 코드를 돌리는 것 이상의 작업임을 체감했다.

**2. 구조적 차이가 실험 시간에서 그대로 드러남**

PatchCore는 학습이 없어 수분 만에 끝나는 반면, SimpleNet은 Discriminator 학습이 필요해 160 epoch 기준 약 6시간이 소요됐다. 논문에서 이론으로 배운 두 모델의 구조적 차이를 실험 시간으로 직접 체감할 수 있었다.

**3. 카테고리에 따라 성능이 갈린다**

hazelnut에서는 두 모델이 I-AUROC 1.000으로 동등했지만, screw에서는 PatchCore 1.000 vs SimpleNet 0.895로 큰 차이가 발생했다. 이는 screw처럼 결함 패턴이 미세하고 다양한 카테고리에서는 메모리 뱅크 기반의 PatchCore가 더 안정적일 수 있음을 시사한다.

**4. SimpleNet의 학습 불안정성**

SimpleNet의 I-AUROC는 epoch마다 0.4~0.9 사이를 오가며 수렴이 불안정했다. 이는 Gaussian noise 기반의 가짜 불량 생성이 screw처럼 복잡한 결함 패턴을 커버하기에 한계가 있을 수 있음을 보여준다. 반면 PatchCore는 별도 학습 없이도 안정적으로 높은 성능을 보였다.

**5. 속도와 정확도의 트레이드오프**

SimpleNet이 논문에서 강조한 77 FPS라는 추론 속도 우위는 실제 산업 현장 적용 측면에서 의미가 있다. 그러나 학습 시간과 카테고리별 성능 편차를 고려하면, 안정성이 중요한 환경에서는 PatchCore가, 실시간 추론이 중요한 환경에서는 SimpleNet이 더 적합할 수 있다는 결론을 도출할 수 있다.

**시각화 결과 분석**

히트맵을 보면 두 모델의 차이가 명확하게 드러난다.

PatchCore는 나사의 결함 위치를 비교적 뾰족하게 잡아낸다. 밝은 영역(흰색~노란색)이 나사 몸체의 특정 부위에 집중되어 있어, 어느 부분이 이상한지를 상대적으로 명확히 표시한다.

반면 SimpleNet은 히트맵이 전체적으로 붉게 퍼져 있어 특정 위치를 잡아내지 못하고 있다. Overlay를 보면 나사 전체가 붉게 표시되는데, 이는 정상과 불량을 구분하지 못하고 나사 자체를 통째로 이상 영역으로 판단하는 것처럼 보인다.

**screw에서 SimpleNet이 잘 안 나오는 이유**

논문에서도 screw는 SimpleNet이 상대적으로 낮은 성능을 보이는 카테고리다. 이유는 세 가지로 볼 수 있다.

첫째, screw의 결함은 나사산 일부가 망가지거나 헤드 부분에 미세한 흠집이 생기는 방식으로, 정상과 불량의 차이가 극히 미세하다. SimpleNet이 피처 공간에서 Gaussian noise를 더해 가짜 불량을 만드는 방식으로는 이런 미세한 경계를 학습하기 어렵다.

둘째, screw는 이미지마다 나사의 방향과 위치가 다르다. 히트맵을 보면 나사가 대각선, 세로, 가로 등 다양한 각도로 찍혀 있는데, SimpleNet의 Discriminator가 이 다양성을 충분히 소화하지 못한 것으로 보인다.

셋째, 학습 불안정성 문제다. 로그를 보면 loss가 0에 가깝게 수렴해도 I-AUROC가 갑자기 0.4대로 떨어지는 현상이 반복됐다. 이는 Discriminator가 정상 피처와 가짜 불량 피처를 너무 완벽하게 구분해버려서, 실제 테스트 시 일반화가 안 되는 오버피팅 현상으로 해석할 수 있다.
