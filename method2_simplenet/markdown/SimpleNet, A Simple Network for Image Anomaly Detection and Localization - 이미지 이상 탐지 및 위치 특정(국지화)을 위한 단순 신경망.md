### PatchCore가 가진 문제, SimpleNet이 푼 방식

## 정상을 기억해서 비교 

**PatchCore는 "정상 제품 도감"을 만드는 방식이다. 

![[Pasted image 20260430113721.png]]
정상 제품의 패치 피처들을 수집해서 메모리 뱅크를 만들어두고, 새 제품이 들어오면 저장된 피처들과 kNN으로 비교해서 거리가 너무 멀면 불량으로 판단하는 방식.

---

**한계 ①: 메모리 뱅크가 커질수록 느려짐**

정상 이미지가 많아질수록 메모리 뱅크도 커지고, kNN 검색 시간도 길어짐. Coreset으로 압축하긴 하지만 근본적으로 검색 방식 자체가 느려서 실시간 적용이 어려움. (~6 FPS)

**한계 ②: 도메인 편향**

PatchCore가 사용하는 피처는 ImageNet(자연 이미지) 기반. 산업 이미지(나사, 카펫, 병 등)는 자연 이미지와 분포가 달라서 피처가 처음부터 산업 도메인에 최적화되어 있지 않음. 중간층(layer2, 3)을 사용해서 어느 정도 완화했지만 근본적인 해결은 아님.

---

**SimpleNet의 발상 전환**

## 불량을 만들어서 학습

> "메모리 뱅크를 만들 필요가 있나? 차라리 불량 피처가 어떻게 생겼는지 직접 만들어서 판별기를 학습시키면 되지 않나?"

메모리 뱅크 대신 정상/불량을 구분하는 판별기(Discriminator)를 학습시키는 방식. 불량 샘플이 없으니까 정상 피처에 Gaussian noise를 더해서 가짜 불량 피처를 생성해 학습에 활용. 추론 시에는 판별기만 남겨서 forward pass 한 번으로 끝냄. → 77 FPS



## SimpleNet 상세요약

### 1. 문제 정의

산업 현장에서 불량 샘플은 희귀하고 종류도 다양해서 모든 유형을 학습시키기 불가능. 따라서 정상 샘플만으로 이상을 탐지해야 하는 cold-start 문제(사전 불량 데이터 없이 시작하는 상황)를 다룸.

기존 방법들의 한계:

- **재건 기반** (Reconstruction-based) : 모델이 너무 잘 학습되면 불량도 정상처럼 재건해버리는 문제 발생
- **합성 기반** (Synthesizing-based): 이미지 공간에서 불량을 직접 만들면 실제 불량과 동떨어진 비현실적인 샘플이 생성됨
- **임베딩 기반(PatchCore처럼 피처를 저장해두고 비교하는 방식)**: 성능은 좋지만 메모리 뱅크와 kNN(가장 가까운 이웃을 찾는 검색 방식) 때문에 속도가 느리고 도메인 편향 문제 존재

---

### 2. 핵심 아이디어 3가지

> "피처를 타겟 도메인에 맞게 변환하고, (메모리 뱅크 대신) 피처 공간에서 불량을 합성해서, 단순한 판별기로 구분하자"

- 사전학습 피처를 그대로 쓰지 말고 Feature Adapter(피처를 산업 도메인에 맞게 변환하는 얕은 네트워크)로 변환 -> 도메인 편향 해결 
- 이미지가 아닌 피처 공간(이미지를 숫자 벡터로 표현한 공간)에서 불량 합성 → 더 현실적인 경계 표현 가능, 속도 해결 
- 복잡한 통계 알고리즘 대신 단순한 Discriminator(정상/불량을 구분하는 판별기)로 구분

---

### 3. 방법론 (4가지 컴포넌트)

**![[Pasted image 20260428141637.png]]**

#### ① Feature Extractor

- PatchCore와 동일한 방식으로 피처 추출
- WideResNet50(ImageNet으로 사전학습된 CNN 백본)의 layer2, layer3 사용
- Neighborhood aggregation(주변 패치 정보를 평균내서 하나의 대표값으로 합치는 방식, p=3)으로 주변 맥락 포함

#### ② Feature Adapter

- FC layer(입력과 출력을 선형 변환하는 단순한 레이어) 단 1개짜리 얕은 네트워크
- **ImageNet 피처 → 산업 도메인 피처로 변환해서 도메인 편향 해결
- 복잡한 구조보다 단순 FC가 오히려 성능 좋음 → 복잡할수록 오버피팅 위험

 ![[Pasted image 20260428141907.png|526]]

- 어댑터 적용 후 피처 분포가 더 촘촘하게 모이는 걸 시각적으로 확인 가능

#### ③ Anomaly Feature Generator

- **정상 피처에 Gaussian noise(정규분포를 따르는 랜덤 노이즈, σ=0.015)를 더하는 것**이 전부
    - `q_anomal = q_normal + ε`, ε ~ N(0, σ²)
- 이미지가 아닌 피처 공간에서 합성하는 게 핵심
    - 이미지에서 직접 합성하면 실제 불량과 거리가 먼 샘플 생성
    - 피처 공간에서는 정상 경계 바로 바깥을 자연스럽게 표현 가능
- σ가 너무 크면 경계가 느슨해져서 미세한 불량을 놓침, 너무 작으면 학습 불안정
- 학습 시에만 사용하고 추론 시에는 완전히 제거됨

![[Pasted image 20260428142131.png]]
이미지 단위 탐지 정확도 / 픽셀 단위 탐지 정확도 
σ=0.015 근처에서 대부분의 클래스가 가장 높은 성능을 보이는것을 알 수 있음.
#### ④ Discriminator

- **정상 피처와 가짜 불량 피처를 입력으로 받아서 구분하는 법을 익히는 판별기
- 2-layer MLP(여러 개의 FC layer를 쌓은 간단한 신경망)로 정상/불량 판별
- Loss function으로 Truncated L1 loss(너무 확신하는 예측에는 gradient를 주지 않아 오버피팅을 방지하는 손실함수) 사용
    -  일반적으로 사용하는 cross-entropy loss보다 I-AUROC 0.2%, P-AUROC 0.3% 높음

추론 시에는 판별기만 남아서 입력 피처의 이상치 점수를 바로 출력함. 메모리 뱅크나 kNN 검색 없이 forward pass 한 번으로 끝나기 때문에 속도가 훨씬 빠름. (77 FPS)
#### 학습 흐름 정리

```
[학습]
정상 이미지
→ ① Feature Extractor로 패치 피처 추출
→ ② Feature Adapter로 도메인 변환
→ ③ Gaussian noise 추가해서 가짜 불량 피처 생성
→ ④ Discriminator가 정상/불량 구분하도록 학습

[추론]
③ Generator 제거
→ ①②④만으로 forward pass 한 번에 끝
```

**Ablation study(각 구성요소가 성능에 얼마나 기여하는지 실험) 주요 결과:**

- Noise scale(σ): 0.015가 최적값
- 계층 조합: layer2+3 조합이 최적 (PatchCore와 동일)
- 백본 의존성 낮음: ResNet50, WideResNet50 모두 99.6% 달성

### 5. 결론 및 한계

**장점:**

- 메모리 뱅크 없이도 동급 이상의 성능
- 실시간 적용 가능한 속도 (77 FPS)
- 구조가 단순해서 학습과 적용이 쉬움

**한계:**

- Feature Adapter와 Discriminator 학습이 필요해서 PatchCore처럼 완전한 학습 없이 바로 쓰는 방식은 아님
- Gaussian noise로 생성한 가짜 불량이 실제 불량의 다양성을 완전히 커버하지는 못할 수 있음

---

![[Pasted image 20260428145259.png]]
- **원본 이미지** 위에 이상 탐지 결과가 겹쳐져 있음
- **주황색으로 칠해진 부분** → 이상치 점수가 높은 영역 (모델이 불량이라고 판단한 곳)
- **빨간색 경계선** → 실제 정답(ground truth) 불량 위치

즉, 주황색이랑 빨간 경계선이 얼마나 일치하느냐가 모델 성능이다.

**broken_small (작은 파손)** 히트맵에서 병 뚜껑 왼쪽의 미세한 파손 위치가 밝게(흰색) 표시됨. 육안으로 확인하기 어려운 작은 결함도 탐지 가능.

**contamination (오염)** 병 표면 하단의 이물질 위치와 히트맵의 밝은 부분이 잘 일치함.

**good (정상)** 집중된 밝은 부분 없이 전체적으로 고른 분포를 보임. 히트맵이 완전히 어둡지 않은 건 정규화 방식에 의한 것으로 오류가 아님. 불량 이미지들과 비교했을 때 뚜렷한 이상 반응이 없다는 점이 핵심.

**정성적 분석**

히트맵 시각화 결과, 대부분의 불량 유형에서 두 모델은 유사한 탐지 패턴을 보임. 그러나 print(인쇄 글씨) 유형의 불량에서 차이가 두드러짐. PatchCore는 불량 영역을 넓게 포착한 반면, SimpleNet은 글씨 획 단위까지 정밀하게 탐지함. 이는 Feature Adapter를 통한 도메인 특화 피처 변환이 미세한 패턴 탐지에 효과적임을 시사함.
