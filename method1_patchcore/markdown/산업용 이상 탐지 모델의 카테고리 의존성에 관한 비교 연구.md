## 초록

본 연구는 산업용 이상 탐지 분야의 대표적인 세 가지 접근법인 PatchCore [1], SimpleNet [2], Reverse Distillation [3]을 MVTec AD 벤치마크에서 재현하고 비교 분석한다. 세 모델은 모두 정상 데이터만으로 학습하는 Cold-start 조건에서 출발하지만, 임베딩 기반(메모리 뱅크), 합성 기반(가짜 불량 생성), 지식 증류 기반(역방향 복원)이라는 근본적으로 다른 패러다임을 채택한다. 본 실험에서 hazelnut 카테고리에서는 세 모델 모두 I-AUROC 1.000을 달성하여 동등한 성능을 보였으나, screw 카테고리에서 PatchCore가 0.988을 기록한 반면 SimpleNet은 epoch 160 완주 후에도 0.895에 그쳤으며 학습 과정에서 수렴 실패 패턴이 명확히 관찰되었다. 본 연구는 이 현상을 단순한 성능 격차가 아닌 SimpleNet의 구조적 설계 가정과 screw 카테고리의 데이터 특성 간 불일치로 해석하고, 세 가지 가설(고정 σ의 한계, 방향 다양성과 Feature Adapter의 표현력, Discriminator 오버피팅)을 통해 분석한다.

## 1. 서론

### 1.1 연구 배경

산업 현장에서의 이상 탐지는 사전에 불량 샘플을 충분히 확보할 수 없다는 본질적 제약을 안고 있다. 결함은 발생 빈도가 낮을 뿐 아니라 그 형태가 사전에 모두 열거되지 않으며, 이로 인해 지도 학습 방식의 적용이 원천적으로 제한된다 [1, §1]. 이 문제는 Cold-start 이상 탐지로 정의되며, 정상 데이터의 분포만 학습한 모델이 분포 바깥(out-of-distribution) 샘플을 식별하는 형태로 환원된다.

본 연구의 대상인 세 모델은 각각 임베딩 기반, 합성 기반, 재구성 기반의 발전된 형태에 위치한다. PatchCore는 임베딩 기반의 정점에 있으며, SimpleNet은 합성을 피처 공간으로 옮긴 모델, Reverse Distillation은 지식 증류 기반에 역방향 구조와 정보의 단절을 도입한 모델이다.

### 1.2 연구 질문

첫째, 동일한 벤치마크 위에서 세 모델의 성능은 얼마나 카테고리에 의존적인가? 둘째, SimpleNet을 screw 카테고리에 대해 학습했을 때 관찰된 수렴 실패 현상은 모델의 설계 가정과 데이터 특성 간의 구조적 불일치를 드러내는 신호인가?

## 2. 세 모델의 학습 패러다임 및 해석

### 2.1 PatchCore: 비학습 메모리 뱅크 방식

PatchCore는 WideResNet50의 layer2와 layer3에서 추출한 패치 단위 피처를 메모리 뱅크로 구성한다 [1, §3.1]. Greedy Coreset Subsampling을 통해 데이터의 1%만 남겨도 성능을 유지하는데 [1, Table 1], 이는 산업 현장에서의 데이터 효율성을 보여준다. 별도의 학습 과정 없이 '정상을 완벽하게 기억'하여 대조하는 방식이기에 본 실험에서도 가장 안정적인 성능을 보였다.

### 2.2 SimpleNet: 피처 공간에서의 가짜 불량 합성

SimpleNet은 "불량 피처가 어떻게 생겼는지 직접 만들어서 판별기를 학습시키자"는 발상에서 시작한다 [2, §1]. Feature Adapter를 통해 도메인 적응을 수행하고, 정상 피처에 가우시안 노이즈($\sigma$)를 더해 가짜 불량을 생성한다 [2, §3.3]. 추론 시 단일 스트림 구조로 77 FPS의 속도를 확보하지만, 합성된 데이터와 실제 결함 사이의 간극이 성능의 핵심 변수가 된다.

### 2.3 Reverse Distillation: 역방향 복원의 비대칭 구조

RD4AD는 Teacher-Student 모델 간의 표현 불일치를 이용한다. 본 연구는 이 모델의 핵심을 '의도된 정보의 단절'로 해석한다.

- **Skip Connection 부재의 의미:** 일반적인 구조와 달리 skip connection을 제거함으로써, 이상치 정보가 Student로 직접 흐르는 것을 차단한다 [3, §3.2]. 오직 OCBE로 압축된 정상 임베딩만을 통과시켜 Student가 '정상의 기억'만으로 복원을 수행하게 강제한다.
    
- **Max Score 산출:** 픽셀 단위 유사도 차이를 이용해 이상 점수를 산출하는데, 이는 스승의 확신과 제자의 당혹감 사이의 간극을 포착하는 동적인 방식이다.

본 실험에서는 공식 코드를 최신 환경에 맞춰 직접 디버깅하여 실험의 신뢰도를 확보하였다.

- **Library Patch:** Pandas 최신 버전에서 `df.append`가 삭제된 문제를 확인하여 `pd.concat`으로 로직을 수정하였다.
    
- **Data Pipeline:** `--data_path` 인자가 무시되는 버그를 발견하여 `sed` 스트림 편집기로 패치를 적용하였으며, 하드코딩된 `item_list`를 동적으로 수정하여 실험 자동화를 구현하였다.
    
- **Normalization:** `torchvision` 업데이트로 유실된 정규화 속성을 ImageNet 표준값으로 복구하여 피처 추출의 왜곡을 방지하였다.
    

## 4. SimpleNet Screw 수렴 실패 현상 분석

### 4.1 현상 기록

`screw` 카테고리에서 SimpleNet은 epoch 13에서 I-AUROC 0.895로 최고점을 찍은 후, 이후 160 epoch 완주까지 0.4~0.9 사이를 격렬하게 진동하며 수렴에 실패하였다.

### 4.2 구조적 원인 가설 및 분석

1. **Discriminator 오버피팅:** 판별기가 '진짜 불량'이 아닌 합성된 '가우시안 노이즈' 자체에 과적합되었을 가능성이 크다 (§4.4).
    
2. **노이즈 스케일 불일치:** `screw`의 미세한 결함에 비해 $\sigma=0.015$ 노이즈가 너무 커서, 정상 피처의 경계를 불필요하게 확장시킨 것으로 보인다 [2, Fig. 5].
    
3. **시각적 증거:** PatchCore가 결함 위치를 뾰족하게(Sharp) 짚어내는 반면, SimpleNet은 수렴 실패 시 히트맵 전체가 붉게 물드는 현상이 나타났다. 이는 모델이 국소적 특징을 잃고 전체적 분포 편향에 매몰되었음을 시사한다.
    

## 5. 결론

본 연구는 세 가지 모델의 재현 실험을 통해 산업용 이상 탐지 모델이 나아가야 할 실무적 방향성을 다음과 같이 도출하였다.

#### 5.1 학습 비용과 추론 속도의 역설 (The Paradox of Efficiency)

실험 결과, 모델의 '효율성'은 두 가지 측면에서 상충한다.

- **PatchCore**는 학습 과정이 없어 즉각적인 현장 배포가 가능하지만, 추론 시 대규모 메모리 검색(kNN)으로 인해 하드웨어 자원을 지속적으로 소모한다.
    
- **SimpleNet**은 추론 속도가 77 FPS로 매우 빨라 실시간 검사에 최적화되어 있으나, 판별기(Discriminator)를 구축하기 위해 **본 실험 기준 6시간 이상의 긴 학습 시간**을 요구한다. 즉, SimpleNet의 "속도 우위"는 '긴 학습 시간'이라는 기회비용을 지불하고 얻은 '빠른 추론 속도'로 정의되어야 한다.
    

#### 5.2 SimpleNet 수렴 실패의 기술적 시사점

특히 `screw` 카테고리에서 관찰된 SimpleNet의 성능 진동(Oscillation)은 **가우시안 노이즈 기반의 가짜 불량(Synthetic Anomaly)이 지닌 한계**를 시사한다. 학습 시간이 6시간에 달했음에도 수렴에 실패했다는 사실은, 단순히 학습량의 문제가 아니라 **가짜 불량의 분포가 실제 미세 결함의 매니폴드를 대변하지 못함**으로써 판별기가 최적의 결정 경계를 찾지 못했음을 의미한다.

#### 5.3 향후 전망: 복원 가이드 기반의 판별 학습 (Hybrid Approach)

본 연구는 이러한 한계를 극복하기 위해 **RD4AD의 '복원 정밀도'와 SimpleNet의 '추론 효율성'을 결합한 하이브리드 구조**를 제안한다.

1. **데이터 증강의 정교화:** 단순 랜덤 노이즈 대신, RD4AD의 역방향 증류 과정에서 발생하는 '복원 오차(Reconstruction Error)'를 판별기의 학습용 불량 샘플로 활용한다.
    
2. **학습 안정성 확보:** RD4AD가 가이드라인을 제시함으로써, SimpleNet의 판별기가 무의미한 진동 없이 더 빠르게 실제 불량 경계에 수렴하도록 유도할 수 있다.
    
3. **현장 최적화:** 학습 단계에서는 RD4AD를 활용해 고품질의 판별기를 만들고, **실제 공정(Inference) 단계에서는 무거운 복원 모듈이나 메모리 뱅크 없이 가벼운 판별기만 단독 사용**함으로써 '고정밀'과 '초고속'을 동시에 달성할 수 있다.
    

결론적으로, 산업 현장의 요구사항에 따라 **즉각적인 대응이 필요할 때는 PatchCore를, 미세 결함의 정밀한 분석과 빠른 검사 속도가 동시에 요구될 때는 본 연구가 제안한 하이브리드 최적화 모델**이 가장 유망한 솔루션이 될 것이다.
