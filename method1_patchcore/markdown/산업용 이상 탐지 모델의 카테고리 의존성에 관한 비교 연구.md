## 초록

본 연구는 산업용 이상 탐지 분야의 대표적인 세 가지 접근법인 PatchCore [1], SimpleNet [2], Reverse Distillation [3]을 MVTec AD 벤치마크에서 재현하고 비교 분석한다. 세 모델은 모두 정상 데이터만으로 학습하는 Cold-start 조건에서 출발하지만, 임베딩 기반(메모리 뱅크), 합성 기반(가짜 불량 생성), 지식 증류 기반(역방향 복원)이라는 근본적으로 다른 패러다임을 채택한다. 본 실험에서 hazelnut 카테고리에서는 세 모델 모두 I-AUROC 1.000을 달성하여 동등한 성능을 보였으나, screw 카테고리에서 PatchCore가 0.988을 기록한 반면 SimpleNet은 epoch 160 완주 후에도 0.895에 그쳤으며 학습 과정에서 수렴 실패 패턴이 명확히 관찰되었다. 본 연구는 이 현상을 두 가지 가설로 분석한다: (1) Discriminator 오버피팅, (2) Anomaly Score 산출 방식의 차이가 localization 정밀도에 미치는 영향. 아울러 세 모델 각각의 구조적 한계를 정리하고 향후 연구 방향을 제시한다.

## 1. 서론

### 1.1 연구 배경

산업 현장에서의 이상 탐지는 사전에 불량 샘플을 충분히 확보할 수 없다는 본질적 제약을 안고 있다. 결함은 발생 빈도가 낮을 뿐 아니라 그 형태가 사전에 모두 열거되지 않으며, 이로 인해 지도 학습 방식의 적용이 원천적으로 제한된다 [1, §1]. 이 문제는 Cold-start 이상 탐지로 정의되며, 정상 데이터의 분포만 학습한 모델이 분포 바깥(out-of-distribution) 샘플을 식별하는 형태로 환원된다.

이를 해결하기 위한 접근은 크게 세 갈래로 나뉜다. **임베딩 기반** 방법은 정상 피처를 저장해두고 거리 측정으로 이상을 판단한다. **합성 기반** 방법은 인위적으로 불량을 생성하여 판별기를 학습시킨다. **지식 증류 기반** 방법은 Teacher-Student 구조에서 표현 불일치를 이상의 근거로 삼는다. 본 연구의 대상인 세 모델은 각각 이 세 갈래의 대표적 구현이다.

### 1.2 연구 질문

첫째, 동일한 벤치마크에서 세 모델의 성능은 얼마나 카테고리에 의존적인가? 논문에서 보고되는 평균 수치 뒤에 가려진 카테고리별 편차는 어떤 구조적 요인에서 비롯되는가?

둘째, SimpleNet을 screw 카테고리에 대해 논문 기본값(epoch 160)으로 학습했을 때 관찰된 수렴 실패 현상은 우연한 학습 사고인가, 아니면 모델의 설계 가정과 데이터 특성 간의 구조적 불일치를 드러내는 신호인가?

## 2. 세 모델의 학습 패러다임

### 2.1 PatchCore: 비학습 메모리 뱅크 방식

PatchCore는 WideResNet50의 layer2와 layer3에서 추출한 패치 단위 피처를 메모리 뱅크 M으로 구성한다 [1, §3.1]. 학습이 필요 없으며, Greedy Coreset Subsampling을 통해 메모리 뱅크를 10%로 압축해도 I-AUROC 99.0%를 유지한다 [1, Table 1]. 이상 점수는 테스트 패치와 메모리 뱅크 간 최대 kNN 거리로 정의되며 [1, §3.3, Eq. 6], 패치별 거리가 공간 위치에 그대로 매핑되어 이상 맵이 생성된다. 본 실험에서 학습 시간이 수 분에 그쳤으며, hazelnut과 screw 모두 안정적인 성능을 보였다.

### 2.2 SimpleNet: 피처 공간에서의 가짜 불량 합성

SimpleNet은 정상 피처에 Gaussian noise (σ=0.015)를 더해 가짜 불량을 생성하고, 이를 구분하는 Discriminator를 학습시킨다 [2, §3.3]. Feature Adapter(단일 FC 레이어)로 ImageNet 도메인 편향을 줄이며 [2, §3.2], 추론 시 Anomaly Feature Generator를 제거한 단일 스트림 구조로 77 FPS를 달성한다 [2, §4.6]. 본 실험에서 epoch 160 기준 약 6시간의 학습 시간이 소요되었으며, PatchCore와의 구조적 차이가 학습 시간으로 직접 체감되었다.

### 2.3 Reverse Distillation: 역방향 복원의 비대칭 구조

Reverse Distillation은 기존 지식 증류 기반 방법의 non-distinguishing filter 문제를 해결하기 위해 Teacher(Encoder)와 Student(Decoder)의 구조를 역방향으로 설계한다 [3, §1]. Student는 Teacher의 출력을 직접 받지 않고, OCBE(One-Class Bottleneck Embedding)를 통해 압축된 정상 표현만을 입력으로 받는다 [3, §3.2]. Skip connection을 의도적으로 제거하여 이상 정보가 Student로 흘러드는 것을 차단하며, 이상 점수는 Teacher와 Student의 피처 맵 간 코사인 유사도 차이로 산출된다 [3, §3.1, Eq. 1]. 본 실험에서 hazelnut 카테고리 기준 I-AUROC 1.000, P-AUROC 0.989를 달성하였다.

### 4.3 시각화 비교

히트맵 시각화에서 두 모델의 차이가 수치 이상으로 명확히 드러났다. PatchCore는 결함 위치(나사산, 헤드 흠집)에 밝은 영역이 뾰족하게 집중되었다. 반면 SimpleNet은 나사 전체에 붉은 영역이 퍼져 있어 결함의 위치를 특정하지 못했다.

이 차이는 anomaly_pixel_auroc 수치에서도 확인된다. full_pixel_auroc 격차(0.995 vs 0.977)보다 anomaly_pixel_auroc 격차(0.994 vs 0.875)가 훨씬 크다. 이는 SimpleNet이 정상/불량 이미지 분류에서는 어느 정도 작동하지만, **불량 영역 내부에서 실제 결함 위치를 픽셀 단위로 특정하는 능력이 특히 부족하다**는 것을 의미한다.

## 5. SimpleNet screw 수렴 실패 분석

### 5.1 가설 1: Discriminator 오버피팅

본 실험 로그에서 loss가 0에 가깝게 수렴한 이후에도 I-AUROC가 0.4대로 급락하는 현상이 반복되었다. loss가 줄어드는데 검증 지표가 폭락하는 이 패턴은 전형적인 오버피팅의 형태다.

SimpleNet은 이를 방지하기 위해 Truncated L1 loss의 truncation term (th⁺=0.5, th⁻=−0.5)을 도입한다 [2, §3.5, Eq. 7]. 이 term은 Discriminator 출력이 임계값을 넘어가면 gradient를 0으로 만들어 확신을 강화하지 못하게 한다. 그러나 본 실험은 이 메커니즘이 screw에서 충분히 작동하지 않음을 보여준다.

Discriminator는 정상 피처와 σ=0.015의 Gaussian noise로 만들어진 가짜 불량 피처를 truncation 임계값에 도달하기 전에 이미 충분히 구분해낸다. 결과적으로 학습이 진행될수록 Discriminator는 학습 데이터 내 정상과 가짜 불량의 경계를 점점 더 정교하게 다듬지만, 그 경계는 실제 screw 결함이 위치한 공간과 점차 어긋난다. **가짜 불량의 분포가 실제 결함의 매니폴드를 대변하지 못함으로써 판별기가 최적의 결정 경계를 찾지 못한 것이다.**

반면 PatchCore는 학습 자체가 없어 오버피팅 문제에서 자유롭다. screw의 다양한 방향과 결함 패턴이 그대로 메모리 뱅크에 반영되어 안정적인 탐지가 가능했으며, 이것이 두 모델 간 screw 성능 차이의 근본 원인이다.

### 5.2 가설 2: Anomaly Score 산출 방식의 차이가 Localization 정밀도에 미치는 영향

본 실험에서 PatchCore와 SimpleNet의 anomaly_pixel_auroc 격차(0.994 vs 0.875)가 full_pixel_auroc 격차(0.995 vs 0.977)보다 현저히 크다는 점은 단순한 성능 차이가 아니라 두 모델의 이상 점수 산출 방식의 구조적 차이에서 비롯된다.

PatchCore는 패치별 kNN 거리를 공간 위치에 직접 매핑하여 이상 맵을 생성한다 [1, §3.3]. 각 위치에서 "이 패치가 메모리 뱅크의 정상 패치들과 얼마나 다른가"를 독립적으로 계산하므로, 결함이 있는 위치에서만 높은 점수가 나온다. 이 구조는 localization 정밀도를 자연스럽게 보장한다.

반면 SimpleNet의 Discriminator는 전체 피처 분포를 기반으로 정상/불량을 판단하도록 학습된다 [2, §3.4]. screw처럼 이미지마다 나사의 방향이 대각선, 세로, 가로 등으로 다양한 카테고리에서는, Discriminator가 개별 결함 위치보다 "나사 전체의 분포"에 반응하는 경향이 생길 수 있다. §4.3의 히트맵에서 나사 전체가 붉게 퍼지는 양상이 이 해석과 일치한다.

즉, **kNN 거리 기반의 위치별 독립 계산(PatchCore)은 결함 위치를 보존하는 반면, Discriminator 출력 기반의 전역적 판단(SimpleNet)은 결함 위치가 분포 전체에 희석될 수 있다.**

## 6. 세 모델의 구조적 한계

### 6.1 PatchCore의 한계

PatchCore의 가장 명확한 한계는 **추론 속도와 메모리 확장성**이다. kNN 검색 기반 추론은 메모리 뱅크 크기에 비례하여 느려지며, 기본 설정 기준 약 6 FPS로 SimpleNet(77 FPS) 대비 현저히 느리다 [1, §4.3]. 데이터가 늘어날수록 메모리 사용량도 증가하며, Coreset Subsampling으로 완화하더라도 근본적 제약은 남는다.

또한 ImageNet 사전학습 피처를 그대로 사용하므로, 산업 도메인과의 분포 불일치가 발생할 수 있다 [1, §1]. SimpleNet의 Feature Adapter나 RD의 OCBE처럼 도메인 적응 메커니즘이 없다는 점에서, 자연 이미지와 분포가 크게 다른 산업 카테고리에서는 성능 저하 가능성이 있다.

### 6.2 SimpleNet의 한계

SimpleNet의 핵심 한계는 **카테고리별 학습 안정성**이다. 본 실험에서 screw 카테고리에 대해 논문 기본값(epoch 160)으로 학습했음에도 수렴에 실패했으며, 이는 고정된 σ=0.015의 Gaussian noise가 모든 카테고리의 결함 특성을 균등하게 커버하지 못함을 시사한다. 산업 현장 적용 시 카테고리별 사전 검증과 σ 튜닝이 필요하다.

또한 epoch 160 기준 약 6시간의 학습 시간은, 논문이 강조하는 77 FPS 추론 속도 우위가 긴 학습 시간이라는 기회비용을 전제로 한다는 점을 보여준다. **SimpleNet의 "추론 속도 우위"는 "긴 학습 시간"이라는 기회비용을 지불하고 얻은 것으로 재정의되어야 한다.**

### 6.3 Reverse Distillation의 한계

Reverse Distillation은 논문에서 직접 언급한 바와 같이 **맥락적(contextual) 결함**에서 성능이 저하된다 [3, §4.1 Limitations]. transistor 카테고리에서 모델은 부품이 어긋난 위치만 표시하지만, 정답은 원래 위치와 새 위치 둘 다를 포함하여 불일치가 발생한다. 이는 OCBE가 단순 파손은 잘 걸러내지만, 공간적 맥락 정보를 요구하는 결함에는 한계가 있음을 시사한다.

또한 Student Decoder를 Teacher Encoder와 대칭으로 직접 설계해야 하는 구현 복잡도가 PatchCore, SimpleNet 대비 높으며, 본 실험에서도 RD4AD 공식 코드의 데이터 경로 하드코딩, pandas 호환 문제 등 다수의 수정이 필요했다.

## 7. 결론

### 7.1 학습 비용과 추론 속도의 역설

세 모델의 효율성은 두 가지 측면에서 상충한다. PatchCore는 학습 없이 즉각적인 배포가 가능하지만 추론 시 kNN 검색으로 하드웨어를 지속 소모한다. SimpleNet은 77 FPS의 빠른 추론이 가능하지만 6시간의 학습 시간을 전제로 한다. Reverse Distillation은 352MB의 낮은 메모리와 0.31s/img의 빠른 추론을 동시에 달성하지만 구현 복잡도가 높다 [3, Table 3].

### 7.2 카테고리 의존성의 시사점

본 연구의 핵심 발견은 세 모델의 평균 성능은 비슷하지만, **카테고리별 안정성은 학습 패러다임에 따라 크게 다르다**는 것이다. 학습이 없는 PatchCore는 카테고리 다양성에 가장 강건하며, 학습 기반의 SimpleNet은 데이터 특성과 하이퍼파라미터의 불일치에 취약하다. 산업 현장에서 모델을 선택할 때 평균 벤치마크 수치만으로 판단하는 것은 위험하며, 실제 적용 카테고리에서의 개별 검증이 필수적이다.

### 7.3 향후 연구 방향

본 연구에서 제안한 두 가설(Discriminator 오버피팅, Localization 방식의 구조적 차이)은 향후 다음과 같은 실험으로 검증될 수 있다. 첫째, screw 카테고리에 대한 σ ablation 실험으로 카테고리 특이적 최적 σ를 탐색한다. 둘째, Reverse Distillation을 screw 카테고리에 적용하여 코사인 유사도 기반 localization이 kNN 거리 기반과 어떻게 다른지 비교한다. 셋째, RD4AD의 복원 오차를 SimpleNet의 판별기 학습용 불량 샘플로 활용하는 하이브리드 구조를 탐색한다.
