# PatchCore vs SimpleNet vs Reverse Distillation

세 모델 모두 "정상 데이터만으로 불량을 어떻게 찾아낼 것인가(Cold-start, 사전에 불량 데이터 없이 시작하는 상황)?"라는 같은 문제를 풀지만, 접근 방식이 완전히 다르다.

- **PatchCore (정상 도감 암기 방식)**
    
    - "정상 제품의 패치(이미지를 작은 조각으로 나눈 것) 조각들을 요약 노트(Coreset, 전체 피처를 대표하는 핵심 샘플 묶음)로 만들어 전부 기억해두자."
        
    - 새 이미지가 들어오면 요약 노트와 일일이 거리를 재서(kNN, 가장 가까운 이웃을 찾는 검색 방식) 다르면 불량으로 판정.
        
- **SimpleNet (가짜 불량 생성 방식)**
    
    - "정상 피처(이미지를 숫자 벡터로 표현한 것)에 노이즈(랜덤 잡음)를 섞어서 인위적으로 가짜 불량을 만들자."
        
    - 메모리를 뒤질 필요 없이, 판별기(Discriminator, 정상/불량을 구분하도록 학습된 신경망)가 이 가짜 불량과 진짜 정상을 구분하도록 학습시켜 한 번에 판별.
        
- **Reverse Distillation (거꾸로 복원 방식 - 현재 논문)**
    
    - "제자(Student) 모델에게 스승(Teacher)이 압축한 요약본만 주고 거꾸로 원본을 그리게 하자."
        
    - 불량 이미지가 들어와도 제자는 '정상'으로만 복원하도록 훈련되었기 때문에, 스승의 눈(있는 그대로 봄)과 제자의 눈(정상으로만 복원함)의 차이를 비교해 틀린 그림을 찾음.

# Reverse Distillation: 거꾸로 복원하며 이상 탐지

**기존 지식 증류(Knowledge Distillation, KD, 잘 학습된 큰 모델의 지식을 작은 모델에게 전달하는 학습 방법)는 스승과 제자의 구조가 똑같다.**

정상 이미지를 넣고 제자가 스승의 반응을 그대로 복사하도록 학습시키는 방식.

기존 방식 (a) ![[Pasted image 20260506174513.png|359]] 
Teacher와 Student 둘 다 위→아래 방향으로 이미지를 압축하는 같은 구조. 구조가 너무 비슷해서 불량 이미지에도 비슷하게 반응해버리는 게 문제.

---

**한계 ①: 불량까지 똑같이 베껴버림 (Non-distinguishing filters, 구분이 안 되는 필터 문제)**

구조가 너무 비슷하다 보니, 처음 보는 '불량' 데이터가 들어와도 제자가 스승의 당황한 반응을 똑같이 흉내 내버림. 결국 둘 사이에 '차이(Discrepancy, 이상 탐지의 근거가 되는 Teacher-Student 반응 차이)'가 발생하지 않아서 뭐가 불량인지 찾을 수가 없음.

**한계 ②: 단조로운 데이터 흐름**

둘 다 이미지를 처음부터 끝까지 똑같은 순서로 훑기 때문에, 다양한 결함을 잡아내는 표현력이 부족함.

---

**Reverse Distillation의 발상 전환**

## 좁은 문을 통과시켜 거꾸로 그리게 하기

> "제자가 스승을 맹목적으로 따라 하게 두지 마라. 스승은 요약만 하고, 제자는 그 요약본을 받아 거꾸로 원본을 복원하게 만들자!"

스승은 정보를 압축하는 **Encoder(이미지를 점점 작은 숫자 덩어리로 압축하는 네트워크)**, 제자는 정보를 다시 풀어내는 **Decoder(압축된 정보를 다시 펼쳐내는 네트워크)** 로 구조를 완전히 다르게 만듦. 제자는 스승이 넘겨준 아주 작게 압축된 요약본(Embedding, 이미지나 피처를 저차원의 숫자 벡터로 표현한 것)만 보고 "원래 모양이 뭐였지?" 하며 거꾸로 복원해야 함. 불량 데이터는 압축 과정에서 다 걸러지기 때문에 제자는 불량을 복원하지 못하고, 여기서 스승과의 확실한 차이가 발생함.

논문 방식 (b) ![[Pasted image 20260506174606.png]]  Teacher(위→아래, 압축)와 Student(아래→위, 복원)의 흐름이 반대인 것을 확인. 구조가 달라서 이상 반응도 확실히 달라짐.

## Reverse Distillation 상세요약

### 1. 문제 정의

지식 증류(KD) 기반 이상 탐지에서 발생하는 **'이상치 복제(불량을 정상처럼 똑같이 처리해버리는 현상)'** 문제를 해결하고자 함.

기존 방법들의 한계:

- **재건 기반**: 모델이 너무 잘 학습되면 불량도 정상처럼 재건해버리는 문제 발생
- **합성 기반**: 이미지 공간(픽셀 단위의 원본 이미지 수준)에서 불량을 직접 만들면 실제 불량과 동떨어진 비현실적인 샘플 생성
- **임베딩 기반(PatchCore처럼 피처를 저장해두고 비교하는 방식)**: 성능은 좋지만 메모리를 많이 쓰고 느림

---

### 2. 핵심 아이디어 3가지

- **이종 구조(Heterogeneous, 서로 다른 종류의 구조를 조합하는 것)**: 스승(Encoder, 압축)과 제자(Decoder, 복원)의 구조를 정반대로 설계.
    
- **정보 병목 (OCBE 모듈)**: 정상 데이터의 핵심만 아주 좁은 통로로 통과시킴. 불량 노이즈는 이 통로를 통과하지 못하고 버려짐.
    
- **역방향 학습(Reverse Order, 고수준 정보부터 시작해서 저수준 정보로 거꾸로 복원하는 순서)**: 스승이 뽑은 추상적인(고차원) 정보부터 시작해서 세밀한(저차원) 정보로 거꾸로 복원하며 학습.
    

---

### 3. 방법론 (3가지 컴포넌트)

![[Pasted image 20260506175056.png|558]] Teacher Encoder(파란 박스)가 이미지를 압축 → Bottleneck(MFF+OCE)에서 정상 정보만 통과 → Student Decoder(주황 박스)가 거꾸로 복원하는 전체 흐름.

---

**① Teacher Encoder (스승 모델)**

- ImageNet(100만 장 이상의 자연 이미지 데이터셋)으로 사전 학습된 ResNet(깊은 신경망을 안정적으로 학습시키기 위해 잔차 연결을 사용하는 CNN 구조) 계열 백본을 사용하며, 가중치를 고정(Freeze, 학습 중 파라미터가 변하지 않도록 잠그는 것)하여 학습 중 변하지 않게 한다.
    
- 입력 이미지로부터 서로 다른 크기(해상도)의 3가지 특징 맵(Feature Map, 신경망이 이미지에서 추출한 정보를 담은 2차원 배열)을 추출한다.
    
    - layer1: 저수준 — 색상, 엣지, 텍스처 같은 디테일
    - layer2: 중간 수준
    - layer3: 고수준 — 구조적·의미적 정보

---

**② OCBE (One-Class Bottleneck Embedding) 모듈 ⭐**

스승과 제자를 이어주는 핵심 다리 역할. 두 블록으로 구성된다.

**MFF (Multi-scale Feature Fusion, 여러 해상도의 피처를 하나로 합치는 블록)**

- 스승이 뽑은 3가지 특징 맵의 해상도를 하나로 맞춰 융합(concat, 여러 텐서를 채널 방향으로 이어 붙이는 것)한다.
- 이유: Decoder가 저수준 피처까지 복원하려면, 그 정보가 bottleneck 안에 들어있어야 하기 때문.

**OCE (One-Class Embedding, 합쳐진 피처를 더 좁은 공간으로 압축하는 블록)**

- MFF 결과를 학습 가능한 ResBlock(잔차 연결이 있는 레이어 묶음, 깊은 네트워크에서도 학습이 잘 되도록 설계됨)으로 저차원 코드로 압축한다.
- **핵심 원리**: 이 모듈은 오직 '정상' 데이터의 정보만 통과하도록 훈련되므로, 테스트 때 흠집(이상) 정보가 들어와도 통과하지 못하고 차단된다.

> 💡 **왜 skip connection(Encoder와 Decoder 사이를 직접 연결하는 지름길 경로)을 쓰지 않나?**  
> 일반적인 AutoEncoder(이미지를 압축했다가 다시 복원하는 구조)는 skip connection으로 디테일을 살리는데, 여기선 쓰면 안 됨. skip path를 열면 이상 정보도 그대로 Student로 새어나가버려서 — Bottleneck이 걸러내는 의미가 없어짐. 의도적으로 bottleneck만 통과하게 강제하는 것.

---

**③ Student Decoder (제자 모델)**

- 스승 모델(ResNet)의 구조를 정확히 반대로 뒤집은 Decoder 구조를 가진다.
    
    - Teacher의 다운샘플링(conv stride 2, 피처 크기를 절반으로 줄이는 연산) → Student의 Deconvolution(conv의 반대 방향 연산, 피처 크기를 두 배로 키움)으로 대응
- OCBE를 통과한 '압축된 요약본'을 시작점으로 삼아, 스승이 뽑았던 3가지 단계의 특징 맵을 거꾸로 복원해낸다.
    
- 학습 시에는 스승의 특징 맵과 자신이 복원한 특징 맵 간의 **코사인 유사도(Cosine Similarity, 두 벡터가 얼마나 같은 방향을 향하는지 나타내는 값, 1이면 완전히 같고 0이면 완전히 다름)** 를 높이는 방향으로 훈련한다.
    

---

### 4. 학습과 추론 흐름

```
[학습]
정상 이미지
→ ① Teacher Encoder(고정): layer1, 2, 3 특징 맵 추출
→ ② MFF: 세 스케일 특징 맵을 하나로 합침
→ ② OCE: 좁은 bottleneck φ로 압축 (이상 신호 차단)
→ ③ Student Decoder: φ → layer1', 2', 3' 거꾸로 복원
→ Loss: Teacher 특징 맵 vs Student 복원 특징 맵의 코사인 유사도 최대화

[추론]
정상/불량 이미지
→ Teacher: 특징 맵 추출 (불량이면 불량 피처 포함)
→ Bottleneck: 이상 신호 걸러냄
→ Student: 복원 시도 → 불량 영역은 복원 실패
→ 위치별로 Teacher vs Student 유사도 계산 → 낮은 곳이 이상 영역
→ 각 스케일 이상 맵을 원본 해상도로 키워서 합산 → 최종 Anomaly Map
→ 이상 맵의 최댓값 = 이미지 단위 이상 점수 (SAD, Sample-level Anomaly Detection score)
```

이상 점수 계산 원리:

- 유사도가 **낮은 위치** = Teacher와 Student 반응이 크게 다름 = **이상 영역**
- 유사도가 **높은 위치** = 복원 성공 = **정상 영역**
- 이미지 전체 평균이 아니라 **최댓값**을 쓰는 이유: 작은 결함은 평균에 묻혀버리기 때문. "어딘가 하나라도 수상하면 불량"이라는 원칙.

### 6. 결론 및 한계

**장점:**

- 스승-제자의 데이터 흐름과 구조를 역방향(Reverse)으로 뒤집은 기발한 아이디어로 고질적인 '이상치 복제' 현상을 해결
- 메모리 뱅크를 저장할 필요가 없어 메모리 사용량 352MB로 가장 작음
- Forward pass(입력이 네트워크를 통과하는 한 번의 연산) 한 번으로 빠르게 탐지 가능 (0.31s/img)
- 가벼운 백본(ResNet18)으로도 97.9% 달성 → 백본 의존성 낮음

**한계:**

- 트랜지스터(반도체 부품)처럼 완전히 망가진 게 아니라 **위치만 살짝 어긋난 논리적 결함**의 경우, 모델이 정상으로 착각하는 경향이 있음
    - 모델 문제가 아니라 어노테이션(ground truth, 정답 레이블) 기준의 문제: 잘못 올려진 부품에 대해 모델은 새 위치만 표시하지만, 정답은 원래 자리+새 자리 둘 다 표시해서 불일치 발생
    - 더 넓은 수용 영역(Receptive Field, 신경망의 한 뉴런이 입력 이미지에서 참조할 수 있는 영역의 크기)을 가진 더 깊은 레이어의 시맨틱 정보(Semantic, 픽셀 단위 디테일이 아니라 "이게 뭔지"에 해당하는 고수준 의미 정보)를 반영해야 함을 시사
- Decoder를 직접 설계해야 해서 구현 복잡도가 PatchCore, SimpleNet보다 높음

## 문제 / 동기

기존 지식 증류(KD) 기반 이상 탐지 모델들은 스승과 제자의 네트워크 구조가 너무 비슷해서(Non-distinguishing filters, 구분이 안 되는 필터 문제), 불량 데이터가 들어와도 제자가 스승의 틀린 특징까지 똑같이 재현해버리는 문제가 있었음. 이로 인해 정상과 불량의 차이(Discrepancy)를 찾아내기 어려웠음.

## 시도

스승은 Encoder(압축), 제자는 Decoder(복원)를 사용하는 이종(Heterogeneous) 구조로 설계. 제자가 원본을 직접 보는 대신, 스승이 추출한 멀티스케일 특징을 OCBE로 압축한 뒤 거꾸로 복원하도록 역방향 증류(Reverse Distillation)를 구현.

- sh: `method3/source/run_rd_hazelnut.sh`
- 노트북: `method3/source/run_rd_hazelnut.ipynb`
