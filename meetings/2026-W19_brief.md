## 3. 막힌 부분 / 의문점

**SimpleNet screw 성능 저하 원인 분석**

세 가지 가설:
1. **미세 결함 한계**: screw 결함은 나사산 일부 파손, 헤드 미세 흠집 등 정상과의 차이가 극히 작음. Gaussian noise 기반 가짜 불량 생성으로는 이런 미세한 경계 학습이 어려울 수 있음
2. **다양한 방향성**: 이미지마다 나사 방향이 대각선/세로/가로 등으로 다양해서 Discriminator가 이 다양성을 충분히 소화 못한 것으로 추정
3. **오버피팅**: loss가 0에 가깝게 수렴해도 I-AUROC가 갑자기 0.4대로 떨어지는 현상 반복 → Discriminator가 정상/가짜 불량을 너무 완벽하게 구분해버려서 실제 테스트 시 일반화 실패 가능성

**시각화에서 드러난 차이**
- PatchCore: 결함 위치를 뾰족하게 특정 부위에 집중해서 표시
- SimpleNet: 히트맵이 나사 전체에 붉게 퍼져 특정 위치를 잡지 못함 → 나사 자체를 통째로 이상 영역으로 판단하는 것처럼 보임

**환경 관련**
- Colab 무료 GPU는 epoch 160 수준에서 한도 소진 → 이후 실험은 Kaggle으로 전환
- RD4AD 코드는 pandas/Python 버전 호환 문제로 수동 패치 필요했음

## 4. 미팅에서 논의하고 싶은 것

- [ ] SimpleNet screw 성능 저하 원인 — Gaussian noise 기반 anomaly generation이 미세 결함(나사산 파손 등)의 decision boundary를 학습하기에 sigma=0.015가 너무 작은 건지, 아니면 screw 카테고리 자체의 방향 다양성 문제인지 구분이 안 됨. sigma 조정 ablation을 해봐야 할지?
- [ ] 세 모델의 anomaly score 산출 방식 차이가 카테고리별 성능 편차에 미치는 영향 — PatchCore는 kNN 거리 기반, SimpleNet은 Discriminator 출력값, RD는 코사인 유사도로 각각 다름. screw처럼 미세 결함이 많은 카테고리에서 어떤 방식이 더 유리한지 이론적으로 정리해보고 싶음
- [ ] RD의 OCBE(Bottleneck)가 이상 신호를 차단한다는 주장 — hazelnut에서는 재현됐지만, screw처럼 정상/불량 피처 분포가 가까운 카테고리에서도 bottleneck이 효과적으로 작동하는지 실험으로 검증할 필요가 있는지?
- [ ] 다음 단계 방향 — 세 모델을 동일 카테고리(screw)로 통일해서 공정하게 비교하는 게 우선인지, 아니면 새로운 method로 넘어가는 게 나은지
